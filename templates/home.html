so-random

This is my final project for CS50. It allows users to generate random lotto numbers for the Australian power ball draw.

The past years draw results are evaluated and each ball number is catagorised as being either hot or cold relative to the number of times it has been drawn. Users are able to configure their draw criteria, once the numbers are generated they can then use developer tools in their browser to run a formatted script (copied to the clipboard via button click) on the oz-lotteries website to fill the from on the order page.

A video demonstrating it's use can be found here.

The brief: Build an order tracking website for high volume customers initially, with the view to add in repeat order functionality (B2B transactions) in the future.

I was half way through CS50 web (which teaches django), when I got side tracked with an opportunity to work on an embedded project. I have been learning C in depth and the in's and outs of developing embedded applications on microcontrollers from ST Microelctronics over the past few months.

The project in question aims to deploy a meshed network of LoRa connected devices that that are able to detirmine amongst themselves which node should be able to activate its drive motor at any given time, given the state of all of the nodes sensors in the network. Only one device motor may be activated in the network at any given time.
